- [new] Fix detached-descendant FD-seek corruption in scripts/run_skill_smoke_lib.finalize_stream_capture_artifact. After a smoke timeout, the cleanup path SIGTERM/SIGKILLs the process group, then `finalize_stream_capture_artifact` re-writes the artifact via `Path.write_text` (open mode `'w'`, truncates to the new normalized JSON length). When a grandchild survives the kill (e.g. a subagent's claude -p inheriting stdout), it keeps writing at its old FD offset, extending the file past the truncation boundary with zero-fill in between, producing a `<normalized-json>\n\x00…\x00<raw-stream-tail>` artifact (`stop-after-before-polish` 2026-04-27 was 319KB with this exact shape and broke strict `json.load`). Today the partial-stream re-grader works around it via `JSONDecoder.raw_decode`, but the writer should be the fix point. Direction: rotate the artifact path before writing the normalized payload (e.g. write to `tool-use-events.json.tmp` then atomic rename), and/or wait for `os.killpg` + descendant scan to confirm no surviving writer before finalize. (added 2026-04-27)
